# WORKHIVE — 업무 협업 관리 플랫폼 기획 명세

## 1. 프로젝트 개요

선임과 후임 간의 업무 공유, 지시·보고 상호 체크, 달성률 시각화를 목적으로 하는 웹 기반 업무 관리 플랫폼.

- **플랫폼**: React 웹 앱 (브라우저 실행)
- **데이터 저장**: 영구 저장 (Claude Storage / 향후 Firebase 등 DB 연동 고려)
- **조직도 방식**: 자유 배치 — 드래그로 노드 이동, 클릭으로 관계 연결

---

## 2. 핵심 기능 요약

### 2-1. 계정 및 관계 관리
- 회원가입 / 로그인 (이름, 이메일, 비밀번호, 직급)
- 직급 선택: 팀장 / 과장 / 대리 / 주임 / 사원 / 팀원
- 계정 간 **선임 ↔ 후임 관계 설정** (조직도에서 연결)
- 한 사람이 선임이자 후임이 될 수 있음 (중간 관리자 구조 지원)

### 2-2. 조직도 (Org Chart)
- 구성원 카드를 **드래그**해 자유 배치
- **관계 연결 모드**: 선임 클릭 → 후임 클릭 → 화살표 라인 생성
- 라인 중간 × 버튼으로 관계 삭제
- 조직도에서 계정 없이도 구성원 추가 가능
- 노드 위치 영구 저장
- 본인 카드 시각적으로 구분 표시

### 2-3. 업무 관리 (Task Board)
세 가지 유형으로 분류:

| 유형 | 방향 | 설명 |
|------|------|------|
| **지시사항** | 선임 → 후임 | 선임이 후임에게 업무 지시 |
| **보고사항** | 후임 → 선임 | 후임이 선임에게 업무 보고 |
| **공유사항** | 양방향 | 관계 무관하게 공유 |

- 업무 등록 시 **대상자 자동 필터링** (지시→후임 목록만, 보고→선임 목록만)
- 체크박스로 완료 처리 / 완료일 자동 기록
- 업무 삭제 (작성자만 가능)
- 마감일(due date) 설정 옵션

### 2-4. 달성률 대시보드
- **일간 / 주간 / 월간** 기간 탭 전환
- 3가지 지표 카드 (% + 진행 바):
  - 전체 달성률
  - 지시 수행률 (나에게 온 지시 중 완료 비율)
  - 보고 제출률 (내가 올린 보고 중 완료 비율)
- **후임 달성 현황**: 선임 입장에서 후임별 달성률 일괄 확인
- 최근 업무 피드 (최신순 6건)

## 3. 화면 구성

```
┌─────────────┬──────────────────────────────────────────┐
│             │                                          │
│  사이드바   │  메인 콘텐츠 영역                         │
│             │                                          │
│  WORKHIVE   │  [대시보드 / 업무관리 / 조직도]           │
│             │                                          │
│  📊 대시보드 │                                          │
│  ✅ 업무관리 │                                          │
│  🏢 조직도   │                                          │
│             │                                          │
│  ─────────  │                                          │
│  [유저정보] │                                          │
│  선임/후임  │                                          │
│  [로그아웃] │                                          │
└─────────────┴──────────────────────────────────────────┘
```

### 화면 목록
1. **로그인 / 회원가입** — 탭 전환 방식
2. **대시보드** — 달성률 카드 3개 + 후임 현황 + 최근 업무
3. **업무 관리** — 탭(지시/보고/공유) + 업무 목록 + 등록 모달
4. **조직도** — SVG 라인 + 드래그 노드 캔버스

## 4. 데이터 모델

### User
```json
{
  "id": "string",
  "name": "string",
  "email": "string",
  "pass": "string",
  "role": "팀장 | 과장 | 대리 | 주임 | 사원 | 팀원",
  "dept": "string",
  "at": "ISO datetime"
}
```

### Task
```json
{
  "id": "string",
  "type": "inst | rep | shared",
  "title": "string",
  "content": "string",
  "fid": "fromUserId",
  "tid": "toUserId",
  "status": "pending | done",
  "due": "YYYY-MM-DD",
  "at": "ISO datetime (생성일)",
  "doneAt": "ISO datetime | null"
}
```

### Relationship
```json
{
  "id": "string",
  "sid": "seniorUserId",
  "jid": "juniorUserId"
}
```

### OrgPosition
```json
{
  "userId": { "x": number, "y": number }
}
```

## 5. 저장 구조 (Storage Keys)

| Key | 내용 |
|-----|------|
| `wh_u` | 사용자 배열 |
| `wh_t` | 업무 배열 |
| `wh_r` | 관계 배열 |
| `wh_p` | 조직도 위치 객체 |
| `wh_uid` | 현재 로그인 ID |
